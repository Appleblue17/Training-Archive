G. Gellyfish and Priority Queue

题目描述

There is a min-priority queue Q Q Q, initially empty. Gellyfish will perform m m m operations on it. Each belongs to one of the two types:

  • Insertion: Given l , r l, r l,r. Gellyfish will generate a uniformly random integer x x x from the range [ l , r ] [l, r] [l,r] and then insert it into Q Q Q.
  • Extraction: Delete the current minimum element in Q Q Q. It's guaranteed that Q Q Q is not empty.

After all the operations are done, Gellyfish wants you to foretell the expected product of all the remaining elements in Q Q Q, taking modulo 998244353 998244353 998244353.

输入描述

Each test contains multiple test cases. The first line contains the number of test cases t t t ( 1 ≤ t ≤ 200 1 \leq t \leq 200 1≤t≤200). The description of the test cases follows.

The first line of each test case contains two integers m m m and v v v ( 1 ≤ m ≤ 500 1 \leq m \leq 500 1≤m≤500, 1 ≤ v ≤ 500 1 \leq v \leq 500 1≤v≤500).

Then m m m lines follow; the i i i-th contains one or three integers, denoting an operation. If the operation is an insertion, then there will be three integers o = 1 o = 1 o=1, l l l, r r r ( 1 ≤ l ≤ r ≤ v 1 \leq l \leq r \leq v 1≤l≤r≤v), separated by space; if the operation is an extraction, then the input line consists of a single integer o = 2 o = 2 o=2.

It is guaranteed that the sum of m m m over all test cases does not exceed 2000 2000 2000. It's also guaranteed that the priority queue is not empty when performing an extraction operation.

输出描述

For each test case, output a single line containing an integer: the expected product of all the remaining elements in the priority queue, modulo 998244353 998244353 998244353.

备注

In the first example test case, note that the product of all the elements in an empty priority queue is considered as 1 1 1.

In the second example test case, two random integers, each from the range [ 1 , 4 ] [1, 4] [1,4], are put in the priority queue after the first two operations. Then, the smaller one is deleted, making the product of all the remaining elements equal to the larger integer. Therefore, the expected product is 1 16 ⋅ ( 1 + 2 + 3 + 4 + 2 + 2 + 3 + 4 + 3 + 3 + 3 + 4 + 4 + 4 + 4 + 4 ) = 25 8 \frac{1}{16} \cdot (1 + 2 + 3 + 4 + 2 + 2 + 3 + 4 + 3 + 3 + 3 + 4 + 4 + 4 + 4 + 4) = \frac{25}{8} 161​⋅(1+2+3+4+2+2+3+4+3+3+3+4+4+4+4+4)=825​, and 25 8 ≡ 873463812 ( m o d 998244353 ) \frac{25}{8} \equiv 873463812 \pmod{998244353} 825​≡873463812(mod998244353).

In the third example test case, since 3 < 4 3 < 4 3<4 and 4 ≤ 4 4 \leq 4 4≤4, the second inserted integer is always the maximum element in the priority queue. Since eventually the size of the priority queue becomes 1 1 1, it's sufficient to see the expected answer is the expectation of the second inserted number, which is 1 3 ⋅ ( 4 + 5 + 6 ) = 5 \frac{1}{3} \cdot (4 + 5 + 6) = 5 31​⋅(4+5+6)=5.
