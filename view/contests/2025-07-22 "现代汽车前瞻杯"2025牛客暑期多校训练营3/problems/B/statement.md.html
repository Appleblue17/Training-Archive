B. Bitwise Puzzle

题目描述

Yuki gives you three non-negative integers a a a, b b b, and c c c. You can perform the following operations up to k = 64 k = 64 k=64 times:\

  1. a ← a ⋅ 2 a \gets a \cdot 2 a←a⋅2;
  2. b ← ⌊ b 2 ⌋ b \gets \lfloor \frac{b}{2} \rfloor b←⌊2b​⌋;
  3. a ← a ⊕ b a \gets a \oplus b a←a⊕b, where ⊕ \oplus ⊕ denotes the bitwise XOR;
  4. b ← b ⊕ a b \gets b \oplus a b←b⊕a.

Please make a = b = c a = b = c a=b=c within no more than k k k operations, or report that there is no valid solution. It can be proven that under the constraints of the problem, there will be a valid solution that does not exceed k = 64 k = 64 k=64 operations if a valid solution exists.

输入描述

Each test contains multiple test cases. The first line of input contains a single integer t t t ( 1 ≤ t ≤ 10 4 1 \leq t \leq 10^4 1≤t≤104) --- the number of test cases. The description of the test cases follows.

The first and only line of input of each test case contains three integers a a a, b b b, and c c c ( 0 ≤ a , b , c < 2 31 0 \leq a, b, c < 2^{31} 0≤a,b,c<231).\

输出描述

For each test case:

\

If there is no valid solution, output a single line containing an integer  − 1 -1 −1;

Otherwise, output two lines. The first line contains an integer  p p p ( 0 ≤ p ≤ k 0 \leq p \leq k 0≤p≤k), describing the number of operations you performed; the second line contains  p p p integers, each in the range  [ 1 , 4 ] [1, 4] [1,4], indicating the sequence of the indices of the operations you performed.

示例1

输入

4
3 5 6
0 0 1
7 7 7
2 9 4

输出

2
4 1
-1
0

2
1 2

备注

In the first test case, initially a = 3 a = 3 a=3, b = 5 b = 5 b=5, c = 6 c = 6 c=6. After the first operation, a = 3 a = 3 a=3, b = 6 b = 6 b=6, c = 6 c = 6 c=6; after the second operation, a = b = c = 6 a = b = c = 6 a=b=c=6, which meets the requirement.

In the second test case, regardless of how the operations are performed, a = b = 0 a = b = 0 a=b=0, which cannot satisfy a = b = c a = b = c a=b=c.\
