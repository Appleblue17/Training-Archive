F. Flower

题目描述

In front of Yuki is a flower with n n n petals. She will perform several rounds of operations. In each round, she will first pick a a a petals off the flower, and then pick off another b b b petals. If the remaining petals are inadequate, she plucks all. She doesn't stop until no petals remain.

Yuki once promised that she would leave if and only if the last petal she picked belonged to the first a a a petals picked in some round of the operation; otherwise, she would stay. Sympathetic, you want to pick off some number of petals first, but you cannot pick all, to ensure that she stays. You must determine the minimum number of petals you need to pick off; in particular, if she can't stay, output " Sayonara \texttt{Sayonara} Sayonara".\

输入描述

Each test contains multiple test cases. The first line of input contains a single integer t t t ( 1 ≤ t ≤ 100 1 \leq t \leq 100 1≤t≤100) --- the number of test cases. The description of the test cases follows.

The first and only line of input of each test case contains three integers n n n, a a a, and b b b ( 1 ≤ n ≤ 10 9 1 \leq n \leq 10^9 1≤n≤109, 1 ≤ a ≤ 10 9 1 \leq a \leq 10^9 1≤a≤109, 1 ≤ b ≤ 10 9 1 \leq b \leq 10^9 1≤b≤109).\

输出描述

For each test case, output a single line:\

\

示例1

输入

3
1 2 3
10 2 3
11 2 3

输出

Sayonara
0
1

备注

In the first test case, n = 1 n = 1 n=1, a = 2 a = 2 a=2, b = 3 b = 3 b=3. You cannot pick any petals, and Yuki will pick all the petals in the first round, thus leaving. Therefore, the output is " Sayonara \texttt{Sayonara} Sayonara".

In the second test case, it is easy to see that even if no petals are picked off initially, Yuki will end up picking all the petals after two rounds of operations, thus staying, so the answer is 0 0 0.\
